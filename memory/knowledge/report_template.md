# Шаблон аналитической справки дежурной смены

Используется инструментом `build_report_draft`. Изменение шаблона —
только через ревью человека (см. `../../safety/BIBLE.md`, P4).

```markdown
# Аналитическая справка дежурной смены — ЧЕРНОВИК

Дата формирования: <ГГГГ-ММ-ДД>
Сформировано автономным агентом agenda_briefing_agent; требует верификации аналитиком.

## 1. Базовый блок (обязательные показатели)
<все 49 показателей каталога; отсутствующие — «— нет данных —»>

## 2. Динамический блок (включён по наличию данных)
<показатели динамического блока, по которым поступили данные;
при отсутствии — явная фраза об отсутствии данных>

## 3. Сигналы для внимания аналитика
<«горячие» темы: показатели, превысившие пороги, с обоснованием>

## 4. Полнота данных
<отчёт check_completeness: собрано N/49, перечень недостающих>

---
Черновик. Выводы и интерпретация — за аналитиком (human-in-the-loop).
```

## Правила наполнения

1. Базовый блок — всегда целиком, даже с пропусками (пропуск ≠ удаление
   строки, а пометка «— нет данных —»).
2. Динамический блок — только по факту поступления данных; каждое
   включение логируется в `../scratchpad.md` с обоснованием.
3. Пороговые правила для раздела 3 заданы в `mcp/server.py` (`HOT_RULES`);
   их изменение — ревью человека.
4. Заголовок всегда содержит пометку «ЧЕРНОВИК» до верификации аналитиком.
