# Источники данных и окна отсечки

Сведено из критериев отбора каталога `indicators.json`
(источник — `Показатели_для_подготовки_материалов.xlsx`). Агент обязан
собирать значение показателя строго внутри его окна: значение вне окна
считается недействительным.

## Расписание поступления источников

| Время | Источник | Показатели (id) | Окно отсечки |
|-------|----------|-----------------|--------------|
| ~06:10 | письмо: кибербезопасность | 57, 58, 59 | прошедшие сутки |
| ~06:30 | письмо: наличность и УС/ВСП | 8–24 | прошедшие сутки |
| ~07:00–07:30 | письмо: КТД | 26 | прошедшие сутки |
| ~07:05 | письмо: IT-инциденты | 55, 56 | 07:00 пред. дня → 07:00 тек. дня |
| ~08:00 | письмо: инциденты безопасности | 27, 28, 30 | прошедшие сутки |
| 09:00 | сводка Координационного Штаба | 32–54 | 09:00 пред. дня → 09:00 тек. дня |
| 20:00–23:00 | инфопанель (последнее обновление) | 1, 2 | прошедшие сутки |
| по факту | HR-отчёты, сводки по ГОСБ | 60–66 | средние за 2 недели / за пред. день |
| 09:00 | сводка природных/техногенных событий | 67, 68 | 09:00 пред. дня → 09:00 тек. дня |

## Правила сравнения

- Динамика (id 2, 12): в ежедневных материалах — к предыдущим суткам,
  в еженедельных — к прошедшей неделе.
- Средние (id 29, 60, 63, 64, 66): с начала года / за 2 недели — считаются
  расчётно, не приходят в письмах.
- Сальдо (id 15, 18, 21, 24, 25): расчётные величины (взнос − выдача),
  вычисляются из поступивших показателей.

## Дедлайн

Все базовые показатели должны быть собраны к 09:05 (начало
Координационного Штаба). Неполный блок к дедлайну → черновик с пометками
«— нет данных —» и эскалацией к аналитику.
