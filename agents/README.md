# Протокол A2A: взаимодействие агентов (п. 9.3 гайда Ouroboros)

Бизнес-процесс подготовки справки разделён между тремя специализированными
агентами. Каждый агент — отдельный контур со своей ролью, политикой
автономии и набором MCP-инструментов (карточки: `*.agent.json` рядом).

## Схема взаимодействия

```
            источники (почта 06:10–08:00, инфопанель, ленты)
                    │
                    ▼
        ┌───────────────────────┐
        │   collector_agent     │  auto
        │  сбор + окна отсечки  │
        └───────────┬───────────┘
                    │ A2A handoff: наблюдения за сутки + отчёт о полноте
                    ▼
        ┌───────────────────────┐
        │    analyst_agent      │  auto, обоснования → scratchpad
        │ полнота, «горячие»    │
        │ темы, состав блоков   │
        └───────────┬───────────┘
                    │ A2A handoff: состав блоков + сигналы
                    ▼
        ┌───────────────────────┐
        │    editor_agent       │  auto для черновика
        │  сборка черновика по  │
        │  шаблону              │
        └───────────┬───────────┘
                    │ A2A handoff: черновик на верификацию
                    ▼
        ┌───────────────────────┐
        │   АНАЛИТИК (человек)  │  human-in-the-loop
        │ верификация, выводы,  │
        │ утверждение и отправка│
        └───────────────────────┘
```

## Контракт handoff (сообщение между агентами)

```json
{
  "protocol": "A2A",
  "from": "collector_agent",
  "to": "analyst_agent",
  "task_id": "briefing-2026-07-19",
  "payload": {
    "date": "2026-07-19",
    "completeness": {"complete": true, "missing_base": []},
    "observations_ref": "data/observations_2026-07-19.json"
  },
  "requires_confirmation": false
}
```

Правила:

1. Каждый handoff ссылается на `task_id` вида `briefing-<дата>` — сквозной
   идентификатор цикла подготовки справки (пишется в scratchpad).
2. Сообщение `requires_confirmation: true` означает обязательное
   подтверждение аналитиком до продолжения (новая тема в повестку,
   отправка материала).
3. Деградация: если к 09:05 базовый блок не полон, collector всё равно
   делает handoff со списком недостающих показателей — editor помечает их
   в черновике как «— нет данных —», аналитик решает, ждать ли источник.
4. Общего состояния у агентов нет: состояние — только в файлах `data/` и
   `memory/`, все обращения — через MCP-инструменты.
