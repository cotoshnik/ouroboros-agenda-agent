# Автономный агент подготовки аналитических справок и мониторинга приоритетной повестки

Навык автоматизирует ежедневную подготовку аналитических материалов дежурной
смены: мониторинг источников, сбор показателей, контроль полноты данных и
сборку черновика справки. Финальная верификация и интерпретация остаются за
аналитиком (human-in-the-loop).

## 1. AS IS — текущий процесс (вход → шаги → выход)

**Входной контракт:** тема/перечень тем справки, приоритетные направления,
срок подготовки, требуемый формат выходного документа.

**Граф ручных операций (time-per-step, суммарно 135–200 мин на справку):**

| # | Шаг | Исполнитель | Время, мин |
|---|-----|-------------|-----------:|
| 1 | Получение задачи / актуализация темы | аналитик | 5–10 |
| 2 | Ручной мониторинг источников (почта, ленты, инфопанели) | аналитик | 30–45 |
| 3 | Отбор релевантных публикаций и сигналов | аналитик | 20–30 |
| 4 | Извлечение фактов, цифр, показателей | аналитик | 25–40 |
| 5 | Сверка данных между источниками | аналитик | 15–25 |
| 6 | Формирование структуры справки | аналитик | 15–20 |
| 7 | Написание текста и итоговая компоновка | аналитик | 20–25 |
| 8 | Ручная корректировка при смене повестки | аналитик | 5–15 |

**Выходной артефакт:** аналитическая справка / тематический обзор /
мониторинговый материал.

**Болевые точки:** 10–18 человеко-часов рутинной нагрузки в день на команду;
ручная навигация по источникам; повторяемость действий; задержка
актуализации структуры при смене повестки; человеческий фактор при сверке.

## 2. Данные навыка — каталог показателей

Агент оперирует каталогом из **67 показателей**
(`memory/knowledge/indicators.json`, источник —
`Показатели_для_подготовки_материалов.xlsx`):

- **Базовый блок (49 показателей)** — обязателен в каждом материале:
  клиентопоток и его динамика/прогноз, остаток наличности, выдача в УС,
  КТД, инциденты безопасности (в т.ч. социнженерия), инциденты СВО
  (19 показателей, окно 09:00→09:00), IT-инциденты (окно 07:00→07:00),
  инциденты кибербезопасности и DDoS, заболеваемость, природные и
  технологические инциденты.
- **Динамический блок (18 показателей)** — включается при наличии данных
  или изменении повестки: прогноз пиковых дней, сравнение с нормальным
  клиентопотоком, взносы/выдача/сальдо ВСП (RUB/EUR/USD), средние значения
  с начала года и пр.

**Правила отсечки и критерии отбора** (за какие сутки берётся значение,
во сколько приходит письмо-источник) хранятся в поле
`selection_criterion` каждого показателя и сведены в
`memory/knowledge/sources.md`. Агент обязан применять их при сборе:
например, инциденты СВО учитываются с 09:00 предыдущего дня до 09:00
текущего, КТД — за прошедшие сутки по письму ~07:00–07:30.

## 3. TO BE — целевой пайплайн

```
источники (почта, инфопанели, ленты)
        │
        ▼
[1] collect   — MCP-инструменты забирают значения показателей за окно
        │
        ▼
[2] validate  — контроль полноты: все 49 базовых показателей поступили?
                сверка окон отсечки с критериями отбора
        │
        ▼
[3] rank      — ранжирование сигналов, выявление «горячих» тем,
                решение о включении динамического блока
        │
        ▼
[4] draft     — сборка черновика справки по шаблону
                (базовый блок всегда + динамический при наличии данных)
        │
        ▼
[5] review    — HUMAN-IN-THE-LOOP: верификация, правки, интерпретация
        │
        ▼
итоговая аналитическая справка
```

Оркестрация — по протоколу A2A между тремя специализированными агентами
(см. `agents/`): Collector → Analyst → Editor.

### MCP-инструменты (сервер `mcp/server.py`, порт 9999)

| Инструмент | Назначение |
|------------|-----------|
| `list_indicators` | каталог показателей, фильтр по блоку (base/dynamic/all) |
| `get_selection_criteria` | критерии отбора и окна отсечки по источникам |
| `save_observation` | сохранить значение показателя за дату (сбор) |
| `get_observations` | получить собранные значения за дату |
| `check_completeness` | каких базовых показателей не хватает за дату |
| `detect_hot_topics` | сигналы, превышающие пороги → «горячие» темы |
| `build_report_draft` | собрать черновик справки (base + dynamic блоки) |

## 4. Политика автономии

| Зона | Режим |
|------|-------|
| Сбор значений показателей из источников | **авто** |
| Контроль полноты и сверка окон отсечки | **авто** |
| Включение динамического блока в черновик | **авто**, с записью обоснования в scratchpad |
| Выявление «горячих» тем | **авто**, но продвижение в повестку — только после подтверждения аналитиком |
| Генерация черновика справки | **авто** |
| Публикация/рассылка итогового материала | **human-in-the-loop** — запрещено без явного подтверждения аналитика |
| Запросы внешних данных за пределами списка источников | **human-in-the-loop** |

Ограничения: агент не интерпретирует показатели и не формулирует выводы и
рекомендации — это зона ответственности аналитика; агент не изменяет
каталог показателей и шаблон справки без ревью человека (см. `safety/`).

## 5. Целевые метрики навыка (для замера AS IS → TO BE)

- время подготовки типовой справки: −50–70%;
- рутинная нагрузка команды: с 10–18 до 4–8 человеко-часов в день;
- доля автоматически собранного черновика в итоговом документе: 70–85%;
- точность и релевантность материалов: ≥ 70% по экспертной оценке;
- время реакции на новую значимую тему: в пределах текущего цикла сбора.
